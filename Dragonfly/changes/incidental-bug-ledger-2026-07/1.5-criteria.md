# 1.5 — Criteria: the incidental-bug ledger (Theme 3)

All gating unless marked advisory. Frozen at gate 4 (sha256 in `decisions.md`). This is an **additive
feature** change (not behavior-preserving) → the bar is **"the feature works" (conformance) + "no
existing behavior broken" (light regression)**. These are prompt files (position-sensitive) → T3
asserts the change is additive-only, verified mechanically.

---

**T1 — The feature fires, and the boundary holds BOTH directions (gating; BY EXECUTION).** In a hunt
scenario presenting the agent, mid-hunt, with **both** (i) a potential bug **clearly unrelated to the
frozen `S#` set** and (ii) an **in-scope** observation that plausibly bears on the `S#`, an agent
following the amended skill: (a) **records (i) in `incidental-ledger.md`** (what / where / why-suspect);
(b) **does NOT investigate or chase (i)** (log-and-move-on); (c) treats (i) as out-of-scope of the `S#`
diagnosis; **and (d) records (ii) in the `observation-ledger.md`, NOT the parking lot** (round-1 fix —
the reverse-direction gap: an in-scope finding must not be mis-routed to the parking lot). *Verified by:*
a cold behavioral arm given the amended files + the scenario; graded FIRED (all of a–d) / DID NOT FIRE
with a quote. A DID-NOT-FIRE (chases (i), fails to log it, folds it into the diagnosis, **or mis-routes
(ii) to the parking lot**) = blocker.

**T2 — The stage-8 surfacing fires on ANY exit (gating; BY EXECUTION).** In a hunt reaching handoff
with a non-empty incidental ledger, the agent **surfaces the incidental ledger as a parking lot**, kept
**separate** from the diagnosed root cause + its (S#-related) residuals — and this fires on **any**
terminal exit, not only the "found → diagnosis.md" path. *Verified by:* **two** cold arms at stage 8
with a populated incidental ledger — (a) a **"found"** handoff and (b) a **"characterized, not found"**
ending (round-1 fix: the characterized/escalation exit, where a hunt didn't nail its symptom, is where
parked findings matter most and the diagnosis-path-only wiring would have dropped them). FIRED (both) =
the handoff carries the parking lot, distinct from residuals.

**T3 — Additive-only, no existing rule changed (gating; mechanical).** The `git diff` of the touched
files (`SKILL.md`, `stages/stage-2.md`, `stages/stage-8.md`, `METHODOLOGY.md`) is **purely additive** —
no existing rule's wording, meaning, placement, or triggering condition is altered; existing lines are
not reordered or edited (only new lines inserted). The insertion neighbours are checked (position lens)
so no existing rule is diluted or displaced. *Verified by:* mechanical diff review at stage 6 + stage 8
(added lines only; unchanged lines truly unchanged).

**T4 — No regression: existing rules still fire (gating; BY EXECUTION, sampled).** The added text does
not break existing behavior. A sample of high-value existing rules still fires under the amended files:
the **representativeness gate** (stage 1) and **gate-before-present** (stage 3) — the two the Tier-2
restructure harness already exercised. *Verified by:* re-running those 2 arms against the amended files;
both must still FIRE. (Stages 1/3 are touched only if the SKILL discipline line is judged to affect
them; even so, the rules must fire.)

**T5 — Per-stage load budget preserved (gating; mechanical).** Every stage's load (SKILL + stage-N
[+ charter]) stays **≤ ~270 lines** (the restructure's C3a cap). The touched stages (2, 8) + SKILL are
the only ones that grow. *Verified by:* `wc -l` per stage load at stage 8.

**T6 — Live == source (gating; mechanical).** After build + install,
`diff -r ~/.claude/skills/dragonfly/` is clean across the touched files. *Verified by:* diff at stage 8.

**T7 — Cross-file consistency (gating; mechanical).** The incidental ledger is described consistently
wherever it appears (SKILL discipline ↔ stage-2 mechanics ↔ stage-8 surfacing ↔ METHODOLOGY artifacts
list ↔ config `ledgers.dir`) — same name, same "log-and-move-on, don't chase, parking lot, out-of-scope
of S#" semantics. *Verified by:* a consistency sweep at stage 8.

**T8 — The parking lot never blocks the hunt (gating; by inspection + T1 arm).** The incidental ledger
is explicitly a parking lot: logging to it never blocks a gate, never enters convergence-cap accounting,
and is never chased. *Verified by:* the rule's wording says so (inspection) + the T1 arm's (b) shows the
agent does not chase.
