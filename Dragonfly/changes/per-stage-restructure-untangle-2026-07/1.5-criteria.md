# 1.5 — Criteria: dragonfly per-stage restructure (Theme 1) + guarded-change untangle (Theme 2)

All criteria gating unless marked advisory. Frozen at gate 4 (sha256 in `decisions.md`), together
with the C1 grep-derived oracle, the C2 shape for the chosen verification tier, and the final file
set from `2-plan.md` §A.

The change touches a **position-sensitive assembly** (prompt files) → **C2 is the mandatory
behavior-preservation criterion**, verified by execution to the depth of the tier chosen at the spec
gate (see `1-spec.md` "Verification depth"). The **concurrency lens does NOT fire** — the restructure
introduces no new accessor or read-modify-write window over shared mutable state; recorded so the
omission is deliberate. The **position lens DOES fire** (these are prompts; rules move, are repeated
across stages, and four borrowed pieces are forked into local copies).

This run mirrors the **proven** guarded-change restructure (`Guarded_change/changes/per-stage-restructure-2026-07`,
live at `3d6889b`): C1–C8 carry over; **C9–C10 are added for Theme 2** (the untangle).

**C1 — Rule inventory preserved and correctly scoped, vs an independent oracle (gating).**
*Oracle, built at stage 2 and frozen at gate 4:* an enumeration of every atomic normative rule in the
CURRENT `Dragonfly/METHODOLOGY.md` + `SKILL.md`, each with a **governing-stage-set derived
mechanically** from the current source (grep of the stages each rule names / the sections it appears
under), not from the author's judgment. **The oracle explicitly includes the rules dragonfly currently
holds *by reference* to guarded-change** (the charter's four lenses + discipline, the severity model,
the rate-based rubric, the lite definition) — after Theme 2 these must be present *in dragonfly's own
files* (this is where C1 and C9 meet). After build, each rule maps to the new stage files with:
- **present** — wording/meaning unchanged (reorganization, not rewrite);
- **correctly scoped** — appears in **every** stage file in its governing-stage-set (a cross-cutting
  rule in all of them; a stage-specific rule in its file); none silently narrowed or widened;
- **no orphans** — none dropped; no *new* normative rule introduced (Theme-3 behavior is explicitly
  NOT introduced here).
Stage-active rules currently living in "reference"-style sections (the append-only ledger duties;
provenance; trust-before-gate ordering; the depth-check + coverage-sweep) are in the oracle and must
land in their governing stage files, not an always-on bucket.
*Verified by, at stage 8 — checked against BOTH the oracle AND the source (round-3 hardening; the
oracle is the organizing enumeration, the SOURCE is the ultimate ground truth so an oracle gap cannot
hide a dropped rule):*
- **(i) forward, oracle→new-location:** the mapping table resolves every one of the **86** frozen
  oracle rows (present + correctly scoped) — zero unmapped / dropped / reworded / mis-scoped rows;
- **(ii) forward, SOURCE→new-location (the teeth):** a stage-6 cold reviewer walks the **current
  `Dragonfly/SKILL.md` + `METHODOLOGY.md`** rule-by-rule and confirms **every normative rule in the
  source** appears in the built files, correctly scoped — this does **not** depend on the oracle being
  complete, so it catches a rule the oracle itself missed (the recurring round-1/2 failure class);
- **(iii) reverse, new-files→oracle:** scan the new files for any normative rule **absent from the
  frozen oracle** — a rule present in the new structure but not the oracle is either a wrongly-**added**
  rule (the over-fork class: position/concurrency, closed-set) OR a source rule the oracle missed
  (reconcile against (ii)); an addition not traceable to source fails the "no new normative rule" clause.
**All three must be clean.**

**C2 — Behavior preserved (gating; BY EXECUTION to the chosen tier).**
**CHOSEN TIER: Tier 2** (owner ThinkerOfThoughts, 2026-07-06 — see `1-spec.md` (B) + `decisions.md`).
Tiers 1 and 3 are retained below for the record; the frozen C2 is the Tier-2 shape. The tier is
frozen at gate 4. Whichever tier, the pass sense is: **no rule that fires under the current whole-doc
structure fails to fire under the new per-stage structure.**
- **Tier 3 (full A/B battery):** for each high-risk rule, a **stage situation** places a cold agent
  *at the stage the rule governs* with a triggering case, under three conditions — **OLD** (current
  whole doc; baseline must fire, else the situation is a battery defect), **NEW** (router + routed
  stage file(s) only; does the rule still fire?), **BROKEN-NEW** (positive control: the rule deleted;
  must NOT fire, else the battery lacks power). Blind grader per situation; frozen per-rule rubric
  (FIRED / DID NOT FIRE / N-A + required quote). Pass: OLD fires, BROKEN does not, NEW fires, for
  every (situation, rule). High-risk rules: the representativeness gate (1/4); the gate-before-present
  tiers (3/7); triage routing (1/4/7); charter cite-or-it-doesn't-count + clean-factual-earned-with-
  citations (1/4/7); provenance discipline (1/4/7); trust-before-gate ordering (4/5); iteration cap →
  escalate (6); depth-check root-or-relay (7); coverage sweep (7); characterized-not-found bar (7/8).
  *Design (n, noise-handling) set in `2-plan.md` §B if this tier is chosen.* **NB — recorded prior:**
  the gc battery and the flagship-probe both found BROKEN controls **non-discriminating** on these
  skills; choosing Tier 3 accepts it will likely re-land on named-risk-acceptance leaning on C1.
- **Tier 2 (content-primary + targeted behavioral spot-check) — the recommended tier:** C1 is the
  primary preservation evidence; a **small** behavioral check confirms (i) the router resolves and
  each sampled stage is reachable via it, and (ii) the 2–3 most position-sensitive rules — the
  representativeness gate, gate-before-present, the charter's cite-or-it-doesn't-count — **still fire
  under NEW per-stage isolation** (arm given only router + routed stage file). Pass: router resolves
  for every sampled stage; each sampled rule fires under NEW. No BROKEN control (its non-discrimination
  is already established twice; a control that cannot fail is not evidence).
- **Tier 1 (content-only):** C1 + the stage-6 cold review carry preservation; no behavioral arms.
  Recorded as a named-risk-acceptance that no execution evidence exists for per-stage firing.
*Verified by:* per the tier — battery/spot-check fire tables + records in `8-harness.md`, or (Tier 1)
the C1 mapping + stage-6 record plus the recorded acceptance.

**C3 — The attention budget is actually reduced, and per-stage isolation holds (gating).**
The change's whole purpose. Two parts:
- **(a) static:** for every stage N, the content required to execute stage N (thin `SKILL.md` +
  `stage-N.md` [+ any reference file that stage explicitly forces, e.g. `charter.md` at 1/4/7]) is
  **materially smaller** than the current per-stage load (today: `SKILL.md`+`METHODOLOGY.md` = 670
  lines, loaded whole at every stage). Targets, recorded per stage: **lines ≤ ~40% of 670 AND tokens
  ≤ ~45% of the current per-stage token load** (token baseline measured at stage 8 from current files).
- **(b) behavioral (the teeth):** if the chosen tier runs NEW-condition arms (Tier 2 or 3), those arms
  **open only the router + their routed `stage-N.md` (+ a reference file that stage EXPLICITLY names)**
  — measured from their transcript Read calls — and do **not** need to open a *sibling stage* file to
  fire their rule. Opening a stage-declared reference file (e.g. `charter.md`) is allowed; opening a
  sibling *stage* file to behave correctly means that file was not self-contained → finding. *If Tier 1
  is chosen, (b) is unmeasured → recorded residual, folded into the C2 Tier-1 acceptance.*
*Verified by:* (a) per-stage line/token table at stage 8; (b) NEW-arm file-open logs (if applicable).

**C5 — Cross-file consistency (gating; mechanical).** Every rule stated in more than one place agrees
wherever it appears within **its own governing-stage-set** — no drift from the split. The sweep is
**driven by the frozen oracle's cross-cutting list** and respects any **deliberate asymmetry** (e.g.
the charter's coverage-challenge applies to a *full-GC triage* run but NOT to dragonfly's direct
stage-7 / lite passes — that absence is by design, not drift, and must not be "corrected"; the
diagnosis-specific aiming lives in the charter, not the gc source). The sweep also covers any residual
rule-restatement left in the slim `METHODOLOGY.md` vs its stage-file home. *Verified by:* a cross-file
consistency sweep at stage 8 (each repeated rule listed against its oracle stage-set, agreement
confirmed).

**C6 — Router correctness (gating; by execution + mechanical).** The router's per-stage pointers
resolve: for each stage (0a, 0b, 1–9), following the router lands on a file that exists and carries
that stage's duties. *Verified by:* (i) mechanical — every stage the loop names has a file the router
points to; (ii) behavioral — if the tier runs arms, their transcripts show each reaching the correct
stage file via the router (stages without an arm rest on the mechanical check; asymmetry recorded).

**C7 — Per-stage legibility (advisory; human-judged rubric).** Each stage file reads as self-contained
— an agent at stage N, reading only it (+ any reference file it names), has every rule it needs.
*Judge:* owner or a stage-6 cold reviewer. *Scale:* per file, "standalone / needs a sibling / broken."
*Pass sense:* every file "standalone." Advisory — the teeth are C1–C6, C9, C10.

**C8 — Total-size budget (advisory).** Per-stage repetition + the four forked pieces raise total line

**C9 — Theme-2 untangle: dragonfly self-contained and faithful (gating). ⟵ the D-S2 fix.**
Two parts:
- **(a) self-containment & removal:** the four borrowed pieces — **reviewer charter** (four lenses +
  evidence discipline + provenance *record-elements*), **severity model** (blocker/major/minor/nitpick +
  iteration cap), **probabilistic/rate-based rubric** (expected rate shift + run count), and the
  **guarded-change-lite definition** — are present **in full in dragonfly's own files** (charter in
  `stages/charter.md`; severity/rubric/lite in the stage files that use them). De-coupling is confirmed
  by **read-confirm + whole-file read-and-classify, NOT a literal-pattern grep** (round-2 fix — the
  broadened pattern set was still adjacency-anchored and still missed SKILL:107 "guarded-change's
  *unchanged* charter" + METH:394 "inherit *ALL of* guarded-change's"): (i) each of the 4 fork targets
  states its rule in dragonfly's own words, with no "see/inherits/reuse/using guarded-change's
  `<piece>`", "by reference", "Identical to guarded-change", or "referenced from guarded-change"; (ii)
  a case-insensitive grep of **every** `guarded-change` occurrence in `Dragonfly/` is read and each hit
  **classified** as a **KEEP composition mention** (the enumerated C10 set — see `2-plan.md` §B) or a
  surviving rules-dependency defect. **Zero surviving rules-dependency hits (every hit accounted for) =
  pass** — the grep *locates* hits; a human/reviewer read *classifies* them, so no adjacency-pattern
  blind spot can hide a survivor. The **D-11 cross-reference self-check is retired**, replaced by a
  light `forked from guarded-change <rev>` provenance note.
- **(b) faithfulness:** each forked piece is **content-faithful** to the current guarded-change source
  it replaces (`Guarded_change/stages/charter.md` for the charter; the SEV1–4 table in
  `stage-8.md:143-174` + `stage-4/7.md` routing for severity; at base rev `3d6889b`) — a mapping table
  shows same rules / same effect. **Recorded divergences (the only permitted ones):** (1) dragonfly's 5
  diagnosis-aiming bullets appended to the charter; (2) the coverage-challenge *exclusion* for direct
  stage-7/lite passes; (3) **four gc-charter items EXCLUDED from the fork — the position lens, the
  concurrency lens, the gc-run-specific A/B supplementary-context clause** (round-1), **and the
  "reviewer-input-is-a-closed-set" rule** (round-2) — none are current dragonfly behavior (its
  spelled-out provenance, oracle B-PROV-1, is the five record-elements only; a source grep for `closed
  set` / `supplementary` / position / concurrency returns nothing); whether dragonfly *should* gain any
  is a separate additive question, surfaced in `9-report.md`; (4) the intended fact that dragonfly no
  longer *tracks* future guarded-change edits. **Recorded transitivity assumption
  (round-1):** the fork source is gc's *post-restructure* charter, whereas the behavior dragonfly
  preserves was defined by gc's *old monolithic* METHODOLOGY (what SKILL:118 referenced); the gc-old →
  gc-new hop was verified behavior-preserving by gc's own separately-run restructure
  (`Guarded_change/changes/per-stage-restructure-2026-07`, live `3d6889b`) — named here, not re-verified.
*Verified by:* (a) the read-confirm + classified grep (zero surviving rules-dependency hits) + presence
check of the four pieces; (b) the faithfulness mapping table + the recorded divergences, at stage 8.
