# 1.5 — Criteria: uploader-retry-backoff

| # | Criterion | Gating? |
|---|---|---|
| **C1** | A 5xx response triggers a retry; a 4xx does not | gating |
| **C2** | Backoff schedule is honored: 1 s, 2 s, 4 s (±20%), max 3 retries, ≤ 90 s total | gating |
| **C3** | No duplicate uploads: an object is never stored twice, including when a retry follows a request that actually succeeded server-side | gating |
| **C4** | Log volume stays reasonable (one line per retry, not per poll) | advisory — operator convenience, not correctness |
