# 1.5 — Criteria: per-stage restructure of guarded-change (Theme 1) — REVISED after stage-3 round 1

All criteria gating unless marked advisory. Frozen at gate 4 (sha256 in decisions.md).
The change touches a **position-sensitive assembly** (prompt files) → C2 is the mandatory
behavior-preservation criterion, verified **by execution** (METHODOLOGY *Information-
preserving is not behavior-preserving* + *stage 8 in detail*). The **concurrency** lens does
NOT fire (no new accessor over shared mutable state) — recorded so the omission is
deliberate.

Revision drivers (stage-3 round 1, `3-redteam-plan.md`): the design is now **file-per-stage**
(round-1 M1); a real attention-budget criterion is added (round-1 blocker C-2); C1 gets an
independent grep-derived oracle (round-1 F1/F2/L4/M2); C2's battery now reaches the rules
that only fire at gate 4+ by positioning arms at those stages (round-1 blocker L1) and adds
a positive control (round-1 L2/M3).

---

**C1 — Rule inventory preserved and correctly scoped, vs an independent oracle (gating).**
*Oracle, built at stage 2 and frozen at gate 4:* an enumeration of every atomic normative
rule in the CURRENT `METHODOLOGY.md`+`SKILL.md`, each with a **governing-stage-set derived
mechanically** from the current source (grep of the stages each rule names / the stage
sections it appears under), not from the author's judgment. After build, each rule maps to
the new stage files with:
- **present** — wording/meaning unchanged (reorganization, not rewrite);
- **correctly scoped** — appears in **every** stage file in its governing-stage-set (a
  cross-cutting rule in all of them; a stage-specific rule in its file); none silently
  narrowed or widened vs the oracle;
- **no orphans** — none dropped; no *new* normative rule introduced.
Stage-active rules currently in "reference" sections (path-validation → gate 4 + reviewer
spawns; `decisions.md` gate-log → iteration cap + stage-8 disposition) are in the oracle and
must land in their governing stage files, not an always-on bucket.
*Verified by:* the oracle→new-location mapping table, every row resolved, at stage 8. Zero
unmapped / dropped / reworded / mis-scoped rows = pass.

**C2 — Behavior preserved, per rule, at the stage it fires (gating; BY EXECUTION — A/B
battery reaching every high-risk rule).**
For each high-risk rule, a **stage situation** places a cold agent *at the stage the rule
governs* and presents a case that should trigger it. Three conditions per situation:
- **OLD** — agent given the current whole doc (rule in view among everything). Baseline:
  the rule should fire → proves the situation is valid.
- **NEW** — agent given only the router + the routed stage file(s) (per-stage isolation, as
  the filesystem will enforce). Treatment: does the rule still fire?
- **BROKEN-NEW** (positive control, on a representative subset) — the stage file with that
  rule deliberately removed. The battery MUST record it as NOT firing → proves the
  battery+grader can detect an absent rule (calibrates power; round-1 M3).
High-risk rules, each at its stage (covers round-1 L1 gate-4+ reachability, P2/P3/P4
adjacency guards, C-1 label audit): position lens (3); provenance discipline (3/6);
"no issue found" valid + **clean factual/label lens earned with citations** (3);
**label audit — a gating criterion mislabeled advisory is caught** (3); **route on
reviewer severity / no silent demotion** (4); **criteria freeze + hash** (4);
**path validation blocks gate 4** (4); **iteration cap → stop for human on 2nd same-class
bounce** (4/7); **gating-criterion-verified-by-execution, no deferral/proxy** (8);
**`decisions.md` gating-disposition entry** (8); **stage-6 mechanical `git diff` (ST6d) +
spot-verify + stage-6 charter self-containment** (6, via SIT-6 — round-2 CC-1).
*Precedence rules (round-2 P-1):* for SEV3 (route on reviewer severity / no silent
demotion), grading requires the arm to ACTUALLY route on the reviewer's stated severity
against the author's contest (a routing/precedence outcome) — not merely to mention the
rule; a text-presence mention is graded DID NOT FIRE.
*Grading:* one cold grader **per situation (6 total)**, `claude-opus-4-8` (Fable
unavailable), **blind to condition** (given a situation's arm outputs with OLD/NEW/BROKEN
labels stripped, IDs randomized); frozen per-rule rubric (FIRED / DID NOT FIRE / N-A, each with a required quote
from the arm output); the rubric supplies the rule *definition* to the grader without
leaking condition (round-1 A3). Author spot-checks every grade; any dispute that would flip
C2 routes to the **owner** (flip-edge).
*Pass (pre-committed):* for every (situation, rule): OLD fires (else the situation is a
battery defect → rebuild it, not a change defect); BROKEN-NEW does not fire (else the
battery lacks power → rebuild); and **NEW fires**. A rule that fires under OLD but not under
NEW is the behavior regression the restructure risks → **blocker → build**. n and the
noise-handling (any single NEW miss → *investigate + re-run that cell*, not blind auto-bounce;
a confirmed drop is the blocker) are set in `2-plan.md` §B and frozen at gate 4.
*Verified by:* battery execution at stage 8; per-(situation×rule×condition) fire tables +
blind grader records in `8-harness.md`.

**C3 — The attention budget is actually reduced, and per-stage isolation holds in practice
(gating). ⟵ the round-1 C-2 blocker fix.**
The change's whole purpose. Two parts:
- **(a) static:** for every stage N, the content required to execute stage N (thin
  `SKILL.md` + `stage-N.md` [+ any reference file that stage explicitly forces]) is
  **materially smaller** than the current per-stage load (today: `SKILL.md`+`METHODOLOGY.md`
  = 658 lines / ~<measure> tokens, loaded whole at every stage). Targets, both recorded per
  stage: **lines ≤ ~40% of 658 AND tokens ≤ ~45% of the current per-stage token load**
  (round-2 F-1 — tokens, not just lines, are the real budget; the token baseline is measured
  at stage 8 from the current files).
- **(b) behavioral (the teeth):** in the C2 battery, **NEW-condition arms open only the
  router + their routed `stage-N.md` (+ a reference file that stage EXPLICITLY names, e.g.
  the config-contract at stage 0/2, or `charter.md` at stages 3/6)** — measured from their
  transcript Read calls — and do
  **not** need to open **sibling stage files** to fire their rule (the flip side of C2's
  pass). Opening a stage-declared reference file is allowed (matches (a)'s bracket, round-2
  L-2); opening a *sibling stage* file to behave correctly means that stage file was not
  self-contained → finding. If NEW arms routinely load everything anyway, the isolation
  failed and the attention win is not real (the exact round-1 blocker).
*Verified by:* (a) per-stage line/token table at stage 8; (b) the NEW-arm file-open logs from
the C2 battery.

**C4 — Live == source (gating; mechanical).** After build + install,
`diff -r ~/.claude/skills/guarded-change/ <repo source>` is clean across SKILL.md, all
`stages/*.md`, and any reference file. *Verified by:* the `diff` output at stage 8.

**C5 — Cross-file consistency (gating; mechanical).** Every rule stated in more than one
place agrees wherever it appears within **its own governing-stage-set** — no drift from the
split. The sweep is **driven by the frozen oracle's cross-cutting list** (round-2 M-2), and
respects the **deliberate stage-3 ≠ stage-6 charter asymmetry** (CH8/CH9/CH10 present in
stage-3.md, absent from stage-6.md by design — round-2 O-1): that asymmetry is NOT drift and
must not be "corrected." The sweep also covers any residual rule-restatement left in the slim
`METHODOLOGY.md` vs its stage-file home (round-2 A-2). *Verified by:* a cross-file
consistency sweep at stage 8 (each repeated rule listed against its oracle stage-set,
agreement confirmed). The standing self-check consistency criterion, generalized to the file
set.

**C6 — Router correctness (gating; by execution + mechanical).** The router's per-stage
pointers resolve: for each stage, following the router lands the agent on a file that exists
and carries that stage's duties (round-1 C-5). *Verified by:* (i) mechanical — every stage
the loop names has a file the router points to (covers all 10 stages); (ii) behavioral — the
C2 NEW-arm transcripts show each arm reaching the correct stage file via the router. Note
(round-2 CC-3): the behavioral half covers only the situation stages (1.5, 2, 3, 4, 6, 8);
router rows for 0, 1, 5, 7 rest on the mechanical check alone — acceptable given the
mechanical backstop, recorded so the asymmetry is deliberate.

**C7 — Per-stage legibility (advisory; human-judged rubric).** Each stage file reads as
self-contained — an agent at stage N, reading only it, has every rule it needs. *Judge:*
owner or a stage-6 cold reviewer. *Scale:* per file, "standalone / needs a sibling /
broken." *Pass sense:* every file "standalone." Advisory — the correctness teeth are C1–C6;
C3(b) already gates the load-bearing part of "self-contained."

**C8 — Total-size budget (advisory).** Per-stage repetition raises total line count across
the file set on purpose; recorded actual vs the plan's estimate. Advisory — per-stage load
(C3), not total, is the real target.
